# SPEC

## 목적

이 문서는 얼굴 나이/성별 예측 GUI 앱의 기능 요구사항, 스레드 분리 기준, 모델 연결 기준, QA 기준을 정의한다. 구현 담당 에이전트와 QA 담당자가 같은 기준으로 작업하도록 현재 코드 구조를 기준으로 작성한다.

## 버전 범위

현재 버전의 목표는 다음과 같다.

1. PyQt5 GUI에서 카메라 미리보기, 얼굴 감지 상태, 측정 버튼, 결과를 표시한다.
2. 카메라 작업과 모델 추론 작업을 QThread 기반 Worker로 분리한다.
3. `models/Best_Age_Estimate_model_traced.pt` TorchScript 모델을 `InferenceWorker` 추론 경로에 연결한다.
4. 정상 측정 완료 후에도 앱을 재시작하지 않고 다음 측정을 진행할 수 있게 한다.
5. 자동 테스트와 수동 QA 기준을 문서화한다.

## 기능 요구사항

### S1. GUI 실행

- `python -m face_age_gender_predictor.app.main_app` 명령으로 PyQt5 GUI 창이 열린다.
- GUI는 카메라 미리보기, 상태 안내, 측정 버튼, 진행 상태, 결과 영역을 가진다.
- 창 종료 시 카메라와 QThread가 정리된다.
- `main_window.py`는 View 역할을 담당하고, 공식 앱 흐름은 `main_app.py`에서 `SystemController`와 연결될 때 동작한다.

### S2. 카메라 미리보기와 얼굴 준비 상태

- 카메라 Worker는 웹캠 프레임을 지속적으로 읽는다.
- 얼굴이 안정적으로 감지되면 GUI가 준비 완료 상태를 표시하고 측정 버튼을 활성화한다.
- 얼굴이 감지되지 않거나 상태가 불안정하면 측정 버튼을 비활성화한다.
- 여러 얼굴이 감지되는 경우 앱이 선택한 대표 얼굴 기준으로 상태를 판단한다.
- 얼굴 인식이 너무 민감해 사용자가 약간 움직였을 때 즉시 취소되지 않도록 안정화 기준은 실제 사용 가능한 수준이어야 한다.

### S3. 측정 시작과 카운트다운

- 사용자는 측정 시작 버튼으로 측정을 요청한다.
- 측정 중에는 중복 클릭을 막기 위해 측정 버튼을 비활성화한다.
- 카운트다운이 GUI에 표시된다.
- 카운트다운 종료 시점에 얼굴 상태를 다시 확인한다.
- 얼굴이 사라졌거나 bbox가 유효하지 않으면 캡처를 시작하지 않고 실패 메시지를 표시한다.

### S4. 40프레임 캡처

- 캡처 조건이 만족되면 카메라 Worker가 40프레임을 메모리 list로 수집한다.
- 캡처 진행률은 GUI에 전달된다.
- 캡처 완료 후 `frames_ready(frames)` 흐름으로 `SystemController`에 전달한다.
- 캡처 중 오류가 발생하면 GUI에 실패 상태를 표시하고 복구 가능한 상태로 돌아간다.

### S5. 모델 추론

- `InferenceWorker`는 40프레임을 받아 `CNNmodel.predict_frames(frames)`를 호출한다.
- 모델 추론은 GUI MainThread에서 실행하지 않는다.
- 기본 모델 경로는 `models/Best_Age_Estimate_model_traced.pt`이다.
- 모델 파일 없음, 모델 로드 실패, 전처리 실패, 추론 실패는 크래시가 아니라 GUI 오류 또는 실패 result로 전달한다.
- `CNNmodel.py`는 import 시 모델 로드, 샘플 추론, plot 실행이 발생하지 않아야 한다.
- 모델은 필요할 때 lazy load하고, 같은 프로세스 안에서는 캐시해 재사용할 수 있다.

prediction dict 형식:

```python
{
    "age": float,
    "gender": float,
    "age_probs": list[float],
    "gender_confidence": float,
}
```

### S6. 결과 후처리

- `result_processor`는 prediction list를 최종 result dict로 집계한다.
- 유효 prediction이 30개 이상이면 성공으로 처리한다.
- 유효 prediction이 30개 미만이면 실패로 처리한다.

성공 result:

```python
{
    "success": True,
    "age": 24.6,
    "gender": 1,
    "age_probs": [...],
    "gender_confidence": 0.91,
    "valid_count": 35,
    "reason": None,
}
```

실패 result:

```python
{
    "success": False,
    "age": None,
    "gender": None,
    "age_probs": None,
    "gender_confidence": None,
    "valid_count": 27,
    "reason": "valid_count_below_30",
}
```

### S7. 결과 표시와 반복 측정

- 성공 시 GUI는 얼굴 미리보기, 나이, 성별, 성별 신뢰도, 나이 분포를 표시한다.
- 실패 시 GUI는 사용자가 이해할 수 있는 실패 메시지를 표시한다.
- 정상 측정 1회 완료 후 결과 화면은 즉시 지우지 않는다.
- 결과 표시 상태에서 얼굴이 다시 정상 인식되면 측정 시작 버튼만 다시 활성화한다.
- 재활성화된 측정 시작 버튼을 사용자가 누르는 시점에 이전 사진, 그래프, 나이, 성별 표시를 초기화하고 새 측정을 시작한다.
- 재측정 흐름은 앱을 껐다 켜지 않아도 동작해야 한다.

## 비기능 요구사항

- GUI는 카메라 읽기와 모델 추론 중에도 응답 가능해야 한다.
- Worker는 GUI 위젯을 직접 수정하지 않고 signal로 상태와 결과를 전달한다.
- 카메라 자원은 하나의 경로에서만 소유한다.
- 모델 파일, 개인 이미지, 대용량 산출물은 Git에 포함하지 않는다.
- 오류는 콘솔에만 남기지 않고 GUI에도 표시한다.
- 파일 인코딩은 UTF-8을 유지한다.

## QA 시나리오

- 정상 흐름: 얼굴 감지, 측정, 카운트다운, 40프레임 캡처, 추론, 결과 표시
- 얼굴 없음: 측정 버튼 비활성화 또는 측정 거부
- 카운트다운 중 얼굴 이탈: 캡처 중단과 실패 메시지 표시
- 얼굴 미세 움직임: 정상 사용 가능한 수준에서는 즉시 취소되지 않음
- 모델 파일 없음: 추론 실패 메시지 표시
- 유효 prediction 30개 미만: 실패 result 표시
- 중복 클릭: 측정 중 추가 요청 무시
- 반복 측정: 결과 표시 후 얼굴 재인식 시 측정 버튼 재활성화, 버튼 클릭 시 이전 결과 초기화
- 종료: 카메라와 QThread 정상 정리
