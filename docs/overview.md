# Project Overview

## 프로젝트 정체성

이 프로젝트는 PyQt5 GUI에서 실시간 카메라 화면을 보여주고, 얼굴이 준비된 상태에서 사용자가 측정을 요청하면 40프레임을 캡처해 나이와 성별을 예측하는 Python 애플리케이션이다.

최종 사용자는 GUI 안에서 카메라 미리보기, 얼굴 감지 상태, 측정 버튼, 카운트다운, 분석 진행률, 최종 예측 결과를 하나의 흐름으로 경험한다. 콘솔 입력이나 임시 테스트 스크립트는 개발 보조 수단일 뿐, 본 프로젝트 문서는 GUI 앱을 기준으로 작성한다.

## 배경

단일 이미지 한 장만으로 나이와 성별을 추론하면 얼굴 검출 실패, 흔들림, 조명, 표정, 프레임 품질에 따라 결과가 크게 흔들릴 수 있다. 이 프로젝트는 실시간 카메라에서 여러 프레임을 수집하고, 프레임별 예측값을 후처리해 더 안정적인 최종 결과를 제공하는 것을 목표로 한다.

또한 GUI가 멈추지 않도록 카메라 루프, 모델 추론, 후처리 작업을 MainThread 밖에서 실행하고, GUI는 사용자 표시와 입력에 집중하도록 구조를 나눈다.

## 사용자 흐름

```text
1. 사용자가 GUI 앱을 실행한다.
2. 앱이 카메라 미리보기를 표시한다.
3. 카메라 워커가 얼굴을 감지하고 준비 완료 상태를 GUI에 전달한다.
4. GUI가 측정 버튼을 활성화한다.
5. 사용자가 측정 버튼을 클릭한다.
6. GUI가 측정 버튼을 비활성화하고 카운트다운을 표시한다.
7. 카운트다운 종료 시점에 시스템이 얼굴 상태를 재검증한다.
8. 카메라 워커가 40프레임을 캡처한다.
9. 추론 워커가 프레임을 전처리하고 모델 예측을 수행한다.
10. result_processor가 prediction 리스트를 최종 결과로 집계한다.
11. GUI가 나이, 성별, 성별 확신도 또는 실패 메시지를 표시한다.
12. 시스템이 다음 측정을 위해 상태를 초기화하고 버튼을 다시 활성화한다.
```

## 핵심 기능

- PyQt5 기반 GUI
- OpenCV 기반 실시간 카메라 프레임 처리
- 얼굴 감지 상태 안정화
- 측정 버튼과 카운트다운 흐름
- 촬영 직전 얼굴 상태 재검증
- 40프레임 메모리 캡처
- QThread 기반 카메라/추론 작업 분리
- TorchScript `.pt` 모델 기반 나이/성별 추론
- 프레임별 prediction dict 후처리
- 성공/실패 결과 표시와 재시도 흐름

## 현재 구현 기준

현재 저장소에는 GUI에 연결할 수 있는 핵심 부품이 이미 분리되어 있다.

```text
main_window.py / AgeEstimatorWindow
= 업로드된 PyQt5 GUI 프로토타입

CameraDetector
= 카메라 프레임 읽기, 얼굴 감지, 40프레임 캡처

CameraBridgeWorker
= CameraDetector callback을 PyQt signal로 변환

SystemController
= 상태 전이, 카운트다운, 촬영 요청, 추론 워커 실행

InferenceWorker
= `CNNmodel.py`의 앱용 추론 API를 호출해 40프레임 prediction을 생성하고 후처리로 전달

CNNmodel.py
= 얼굴 전처리와 `models/Best_Age_Estimate_model_traced.pt` TorchScript 추론을 위한 앱용 API 제공

result_processor.py
= prediction 리스트 평균 집계
```

업로드된 `main_window.py`는 현재 자체적으로 카메라 시작, 프레임 표시, 얼굴 감지, 40프레임 수집, 임시 나이/성별 추정을 수행한다. 다음 개발 단계는 이 GUI를 현재 시스템에 부착하고, 기존 Worker/QThread 구조와 맞춰 실제 앱 흐름을 봉합하는 것이다.

## 이번 단계의 목표

1. 업로드된 GUI 코드를 현재 시스템에 부착한다.
2. GUI, 카메라, 추론, 후처리 작업이 QThread 기반으로 분리되어 안정적으로 동작하게 한다.
3. GUI 통합 후 정상/실패/종료 흐름을 QA 테스트한다.
